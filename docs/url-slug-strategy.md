# URL Slug Strategy

## Project

Eric Deloach Author Website

## Purpose of This Document

This document defines the URL structure for the Eric Deloach author website.

The goal is to create clean, readable, SEO-friendly URLs that support book discovery, author discovery, series navigation, media engagement, and future website growth.

Claude Code must follow these URL rules unless the user explicitly approves a change.

## Primary URL Strategy

The site should use short, clear, lowercase URLs with hyphens between words.

Preferred format:

```text
/books/rage-of-angels
```

Avoid:

```text
/books/RageOfAngels
/book?id=123
/rage-of-angels-book-by-eric-deloach
/rage_of_angels
```

## Core Slug Rules

All slugs must be:

```text
Lowercase
Readable
Short
Descriptive
Hyphen-separated
Stable
Easy to share
Easy for search engines to understand
```

Do not use:

```text
Capital letters
Underscores
Special characters
Query-string style URLs
Random IDs
Dates unless needed for blog posts
Overly long keyword-stuffed URLs
```

## Individual Book URL Pattern

Approved pattern:

```text

```

This structure should be used for all individual book pages.

```text
It is clear
It groups all books together
It supports future books
It avoids duplicate top-level URLs
It helps search engines understand site hierarchy
It keeps navigation clean
```

```text

```

## Canonical URL Strategy

Each page should have one canonical URL.

Do not create duplicate pages for the same content.

Example:

Use:

```text
/books/rage-of-angels
```

Do not also create:

```text
/rage-of-angels
/rage-of-angels-book
/books/rage-of-angels-eric-deloach
```

The canonical URL should be reflected in metadata and schema.

## Breadcrumb Strategy

Use breadcrumbs for nested pages.

Example for individual book pages:

```text
Home > Books > Rage of Angels
```

Example URL structure:

```text
/ > /books > /books/rage-of-angels
```

Example for series page:

```text
Home > Series > Rage of Angels
```

Example URL structure:

```text
/ > /series/rage-of-angels
```

## SEO Slug Principles

Book title slugs should match the exact book title as closely as possible.

Examples:

```text
Rage of Angels → /books/rage-of-angels
Ascension: The Path to Polaris → /books/ascension-the-path-to-polaris
Sins of the Father → /books/sins-of-the-father
In Harm’s Way → /books/in-harms-way
```

The slug should not be stuffed with author names because the page title and metadata will already include Eric Deloach.

Avoid:

```text
/books/rage-of-angels-eric-deloach-dark-fantasy-book
```

## Page Title Support

URLs should work together with page titles.

Example:

URL:

```text
/books/rage-of-angels
```

SEO title:

```text

```

URL:

```text
/series/rage-of-angels
```

SEO title:

```text

```



Future blog posts should follow:


Future media items, if individual pages are created, should follow:

```text

```

Do not change existing slugs when new books are added.

## Slug Locking Rule

Once a slug is approved and built, do not change it unless absolutely necessary.

Changing slugs can break:

```text
Internal links
External links
Search indexing
Retailer references
Social shares
Bookmarks
Schema references
Sitemap entries
```

If a slug must change, create a redirect plan.

Do not publish unfinished placeholder pages.

If a page is not ready, do not link it or include it in the sitemap.

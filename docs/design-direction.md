# Design Direction

## Project

Eric Deloach Author Website

## Purpose of This Document

This document defines the visual design direction for the Eric Deloach author website.

It should guide layout, mood, typography, color, spacing, imagery, components, motion, and responsive behavior.

Claude Code must follow this design direction while preserving approved content, approved page structure, approved slugs, and static export compatibility.

## Primary Design Goal

The website should feel like a premium author platform for cinematic genre fiction.

The design must support Eric Deloach’s two story lanes:

```text
Epic mythological fantasy through the Rage of Angels universe
Grounded military action through In Harm’s Way
```

The site should feel immersive without becoming confusing, overdesigned, or slow.

Recommended color direction:

```text
Near-black backgrounds
Charcoal panels
Deep blue undertones
Warm ember accents
Muted gold highlights
Steel gray dividers
Soft off-white text
Subtle red or copper tension accents
```

## Suggested Color Roles

## Backgrounds

```text
Near-black for primary backgrounds
Charcoal for elevated panels
Deep blue-black for fantasy sections
Gunmetal gray for tactical sections
```

## Text

```text
Off-white for primary text
Soft gray for body text
Muted gray for metadata and labels
Warm highlight for important words
```

## Accents

```text
Gold or ember for mythological fantasy accents
Steel blue or muted gray for tactical action accents
Copper or red sparingly for danger and intensity
```

## Color Rules

Claude Code must:

```text
Use a consistent dark palette
Keep text readable
Use accent colors sparingly
Create contrast between fantasy and tactical sections
Avoid color overload
```

Claude Code must not:

```text
Use bright cartoon colors
Use random gradients
Use low-contrast text
Use excessive red
Use fake glowing effects everywhere
Make the design look like a game splash screen
```

```text

Keep body text readable
```


Avoid:

```text
Crowded cards
Tight line spacing
Dense walls of text
Overlapping elements
Unclear section breaks
```

## Motion Direction

Motion should be subtle and premium.

Acceptable motion:

```text
Soft fade-in
Small upward reveal
Subtle hover lift
Gentle card shadow change
Button hover transition
```

Avoid:

```text
Excessive animations
Spinning elements
Autoplay motion backgrounds
Flashing effects
Motion that harms readability
Animations required to understand content
```

Respect reduced motion settings.

## Background Texture Direction

Acceptable background treatments:

```text
Soft gradient
Noise texture
Subtle smoke effect
Subtle star field
Thin grid pattern
Dark vignette
```

Avoid:

```text
Busy textures
Unreadable backgrounds
Repeated obvious patterns
Large animated backgrounds
Distracting effects behind text
```


## Accessibility Direction

Design must support accessibility.

Requirements:

```text
Readable contrast
Keyboard focus states
Semantic headings
Alt text for meaningful images
Accessible button labels
Responsive touch targets
No text trapped inside images
No autoplay video
Reduced motion support
```

Do not sacrifice accessibility for cinematic effects.

## Performance Direction

The design should feel premium without becoming heavy.

Performance rules:

```text
Optimize images
Limit homepage embeds
Use static data
Avoid unnecessary dependencies
Avoid large background videos
Avoid excessive animation libraries
Preserve static export compatibility
```

## Static Export Direction

The design must support Cloudflare Pages static export.

Claude Code must:

```text
Keep components static-export friendly
Avoid server-only image behavior
Avoid runtime-only dynamic routes unless generated statically
Ensure /out is generated by build
Use verified local assets or external links
```

```text

```

Claude Code must not adjust:

```text
Approved headlines
Approved body copy
Approved CTAs
Approved slugs
Approved metadata
Approved schema content
Approved book facts
Approved character details
```

# Metadata Plan

## Project

Eric Deloach Author Website

## Purpose of This Document

This document defines the metadata strategy for the Eric Deloach author website.

It includes page titles, meta descriptions, Open Graph titles, Open Graph descriptions, canonical URL rules, social preview guidance, and metadata quality rules.

Claude Code must follow this plan when implementing metadata in Next.js.

## Primary Metadata Goal

Metadata should help search engines, AI systems, social platforms, and readers understand:

```text
Who Eric Deloach is
What books he has written
Where each book fits in his catalog
What the Rage of Angels universe is
What In Harm’s Way is
Where readers can watch trailers and find books
```

## Metadata Tone

Metadata should be:

```text
Clear
Accurate
Search-friendly
Reader-focused
Concise
Specific
Free of fake hype
```

Do not use metadata to make claims that are not verified.

## Metadata Rules

Claude Code must:

```text
Use unique metadata for every core page
Use approved titles and descriptions
Keep metadata factual
Use canonical URLs that match approved slugs
Use verified book titles and author name
Use no em dashes
Avoid keyword stuffing
Avoid fake awards or fake rankings
```

Claude Code must not:

```text
Invent awards
Invent bestseller claims
Invent reviews
Invent rankings
Invent media coverage
Invent retailer availability
Invent prices
Invent release details
Use em dashes in metadata
Use duplicate metadata across pages
Use placeholder metadata
Use vague titles such as Home or Books only
```

## Canonical Author Name

Working canonical author name:

```text
Eric Deloach
```

Known variants that may support search discovery:

```text
Eric DeLoach
Eric De’Loach
ERIC DELOACH
Eric Loko
```

Use the canonical name in primary metadata.

Use variants sparingly and only where relevant.

## Site Name

Recommended site name:

```text
Eric Deloach
```

Optional expanded site name:

```text
Eric Deloach Author Website
```

Brand phrase:

```text
Myth and Mayhem
```

Use “Myth and Mayhem” where it supports brand recognition, not on every page title.

## Domain Placeholder

Until the final domain is confirmed, use:

```text
https://[domain]
```

Replace with the final production domain before launch.

Do not use localhost or a temporary preview URL in final canonical metadata unless explicitly instructed.

## Metadata Length Guidance

SEO titles should usually stay under:

```text
60 characters when possible
```

Meta descriptions should usually stay under:

```text
155 to 160 characters when possible
```

Do not force awkward copy just to meet length guidelines. Clarity matters most.

## Open Graph Strategy

Open Graph metadata should be more emotionally engaging than SEO metadata while staying factual.

Good example:

```text
Enter the worlds of Eric Deloach, where mythology and mayhem collide across dark fantasy, celestial warfare, and gritty military action.
```

Bad example:

```text
Read the bestselling books everyone is talking about.
```

unless verified.

## Canonical Strategy

Every page should have a canonical URL matching the approved slug.

Examples:

```text
https://[domain]/
https://[domain]/about
https://[domain]/books
https://[domain]/books/rage-of-angels
https://[domain]/books/in-harms-way
```

Do not create canonical URLs with:

```text
Tracking parameters
Preview domains
Localhost
Duplicate slug variations
Unapproved trailing slash behavior
```

Match the final Next.js trailingSlash configuration.

## Robots Metadata

Public pages should be indexable when launched.

Default:

```text
index, follow
```

Do not index unfinished pages.

If a page is not ready:

```text
Do not publish it
Do not include it in navigation
Do not include it in sitemap
```

Avoid relying only on noindex for unfinished public pages.

## Metadata Implementation in Next.js

Use Next.js metadata support.

Recommended approach:

```text
Use static metadata exports or metadata helper functions
Keep page metadata close to page data
Use central site config for domain, site name, and default images
Use book data for book page metadata
```

Recommended files:

```text
src/data/site.ts
src/data/books.ts
src/lib/metadata.ts
```

Potential helper:

```text
createMetadata()
```

Metadata should pull from central data files where practical.

Recommended central site data:

```text

socialLinks
```

Recommended book metadata fields:

```text

slug
metaTitle

ogTitle
ogDescription

canonicalUrl
```

## Metadata and Schema Alignment

Metadata and schema should agree.

If metadata says:

```text
Rage of Angels by Eric Deloach
```

Book schema should also use:

```text
name: Rage of Angels
author: Eric Deloach
```

Do not create conflicting names or titles across metadata and schema.

## No Fake Claim Rule

Do not use fake claims in metadata.

Avoid:

```text
Award-winning
Bestselling
Critically acclaimed
Internationally acclaimed
Fan favorite
Viral sensation
Must-read
Top-rated
```

unless verified and approved.

## Metadata Keyword Stuffing Rule

Do not stuff metadata with long keyword strings.

Bad:

```text
Eric Deloach author books Rage of Angels dark fantasy novel Anunnaki Gods audiobook Victor Mason Astrius best fantasy thriller Navy SEAL In Harm’s Way
```

Good:

```text
Explore books by Eric Deloach, author of the Rage of Angels universe and In Harm’s Way. Discover dark fantasy, gritty action, trailers, and retailer links.
```

## No Em Dash Rule

Do not use em dashes in metadata.

Avoid:

```text
Eric Deloach — Author of Rage of Angels
```

Use:

```text
Eric Deloach | Author of Rage of Angels
```

or:

```text
Eric Deloach: Author of Rage of Angels
```

## Metadata QA Checklist

Before launch, confirm:

```text
Every page has a unique SEO title
Every page has a unique meta description
Every page has Open Graph metadata
Canonical URLs match approved slugs
No placeholder domain remains after final domain confirmation
No localhost URLs appear
No preview URLs appear unless approved
No fake awards appear
No fake bestseller claims appear
No fake reviews appear
No fake retailer claims appear
No em dashes appear
Book page metadata matches book page content
Series page metadata matches series content
Social images are approved and available
Social image paths work
Metadata does not keyword stuff
Robots metadata is appropriate
Unfinished pages are not indexed
```

# Book Pages Content Plan

## Project

Eric Deloach Author Website

## Purpose of This Document

This document defines the content structure, page requirements, copy direction, metadata approach, schema needs, and conversion strategy for all individual book pages on the Eric Deloach author website.

Claude Code must follow this structure when building book pages and must not rewrite approved copy unless explicitly instructed.

## Core Book Pages

The initial website should include dedicated pages for:

```text
/books/rage-of-angels
/books/ascension-the-path-to-polaris
/books/sins-of-the-father
/books/in-harms-way
```

## Book Page Purpose

Each book page should:

1. Introduce the book clearly
2. Explain where it fits in Eric Deloach’s catalog
3. Show the book cover
4. Provide approved story copy
5. Present verified book details
6. Include trailer or video embeds when verified
7. Guide readers to related books
8. Provide verified retailer links
9. Support SEO, AEO, schema, and social sharing
10. Avoid unverified claims

## Core Book Page Template

Every book page should follow this structure:

```text
1. Hero
2. Book Details
3. Story Overview
4. Story Hooks or Character Hooks
5. Media Section
6. Series Path or Related Books
7. Reader Fit
8. Retail Links
9. FAQ
```

## Standard Book Page Layout

## 1. Hero

Purpose:

The hero should immediately tell the reader what the book is, where it fits, and why it matters.

Recommended hero fields:

```text
Book cover
Book title
Series or standalone label
Genre label
Short subheadline
Primary CTA
Secondary CTA
Optional media CTA
```

Recommended hero CTAs:

```text
Buy the Book
Listen to the Audiobook
Watch the Trailer
Start the Series
Explore the Universe
Meet Charlie One Alpha
```

Use only CTAs that apply to the specific book.

## 2. Book Details

Purpose:

The Book Details section should provide clear, verified facts about the book.

Potential fields:

```text
Title
Author
Series
Reading order
Genre
Release date
Formats
ISBNs
Page count
Audiobook runtime
Goodreads editions
Retailer availability
```

Use only verified fields.

If metadata varies by retailer, use cautious wording.

Preferred:

```text
Page count varies by format and retailer.
```

Avoid:

```text
This book is exactly 243 pages in every edition.
```

## 3. Story Overview

Purpose:

The Story Overview section should give readers the main narrative hook without spoiling major twists.

The overview should:

```text
Introduce the central character or story world
Clarify the core conflict
Show the emotional stakes
Create reader interest
Avoid major spoilers
Use approved copy only
```

## 4. Story Hooks or Character Hooks

Purpose:

This section should break the book’s appeal into short, skimmable hooks.

Examples for fantasy pages:

```text
A policeman with a celestial bloodline
The Anunnaki Gods
The descent into darkness
Savior or destroyer
Power always comes with a price
```

Examples for military action pages:

```text
The boys of Charlie One Alpha
Tactical survival
Military brotherhood
Loyalty under pressure
Violence and consequence
```

## 5. Media Section

Purpose:

The Media section should use verified trailers, YouTube Shorts, or promotional videos to increase engagement.

Rules:

```text
Only embed verified video URLs
Use responsive video embeds
Do not publish broken embeds
Do not invent video titles
Do not invent trailer lengths
Add accessible labels or headings
```

If video URLs are missing, leave internal TODOs only.

Do not display visible placeholder copy.

## 6. Series Path or Related Books

Purpose:

This section should help readers understand where the book fits and what to read next.

For Rage of Angels universe books, link to:

```text
/series/rage-of-angels
/books/rage-of-angels
/books/ascension-the-path-to-polaris
/books/sins-of-the-father
```

For In Harm’s Way, link to:

```text
/books
/media
/about
```

## 7. Reader Fit

Purpose:

This section should help readers self-identify with the book.

Use language such as:

```text
This book is for readers who love:
```

Keep reader-fit bullets specific and genre-aware.

Do not use fake praise.

Good:

```text
Dark fantasy with moral stakes
Gods, devils, angels, and ancient mythology
Supernatural action and cosmic warfare
Heroes forced to become dangerous
```

Bad:

```text
Readers everywhere are calling this book unforgettable
Critics agree this is a must-read
A bestselling sensation
```

## 8. Retail Links

Purpose:

Retail links should make it easy for readers to choose their preferred retailer and format.

Retail buttons may include:

```text
Amazon
Apple Books
Apple Books Audiobook
Google Play Books
Barnes & Noble
Goodreads
Indigo / Kobo
Booktopia
ThriftBooks
Hoopla through participating libraries
```

Only show buttons verified for that specific book.

Do not show prices unless the client explicitly approves and verifies them before launch.

## 9. FAQ

Purpose:

FAQ sections should support SEO, AEO, reader clarity, and schema.

FAQ topics should be safe, useful, and based on approved content.

Common FAQ topics:

```text
What is this book about?
What genre is this book?
Is this book part of a series?
Where should I start reading?
Is there an audiobook?
Where can I buy the book?
```

Do not use FAQs to add fake facts.

## 10. Final CTA

Purpose:

End each book page with a strong next step.

Examples:

```text
Start the Rage of Angels Series
Buy Rage of Angels
Continue to Ascension
Explore the Rage of Angels Universe
Read In Harm’s Way
Watch the Trailer
View All Books
```

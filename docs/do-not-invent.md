# Do Not Invent

## Project

Eric Deloach Author Website

## Purpose of This Document

This document defines strict content integrity rules for the Eric Deloach author website.

Claude Project, Claude Code, VS Code implementation work, and any future site updates must follow this document.

The website must be built from verified sources, client-approved content, and approved project documents. No fake facts, fake praise, fake metadata, fake awards, or invented story details may appear on the live site.

## Primary Rule

Do not invent anything.

If a fact is not verified, client-provided, or approved in the project documents, do not publish it on the website.

When in doubt, leave it out.

## Approved Source Types

Website content may be based only on:

1. Client-provided author information
2. Client-provided social links
3. Client-provided retailer links
4. Verified retailer pages
5. Verified author profiles
6. Verified book listings
7. Approved project documents
8. Approved final copy documents
9. Direct client instructions

## Not Approved as Sources

Do not use unsupported assumptions from:

* Search snippets without verification
* AI-generated guesses
* Unverified summaries
* Similar author websites
* Similar book descriptions
* Fan assumptions
* Placeholder copy
* Auto-generated metadata
* Fictional reviews
* Fictional praise
* Fictional press blurbs
* Fictional rankings

## Book Fact Rules

Book facts must match verified or approved sources.

Do not invent:

* Release dates
* Page counts
* ISBNs
* Edition counts
* Audiobook availability
* Audiobook runtime
* Format availability
* Retailer listings
* Series placement
* Reading order
* Genre labels
* Subtitle text
* Publisher names
* Awards
* Sales performance

If page counts differ by retailer, use cautious wording.

Preferred wording:

```text
Page count varies by format and retailer.
```

Avoid:

```text
This book is exactly 243 pages in every edition.
```

## Retailer Rules

Do not display a retailer button unless the exact book-specific link is confirmed.

Do not claim a title is available from a retailer unless that specific title is verified for that retailer.

Do not hard-code prices unless the client explicitly requests it and the price is verified right before launch.

Preferred wording:

```text
Available in multiple retailer-supported formats.
```

Avoid:

```text
Available everywhere now for $14.99.
```

## Price Rules

Retail prices can change.

Do not publish prices unless:

1. The client explicitly requests visible pricing
2. The price is verified immediately before launch
3. The page includes a note that prices may vary by retailer

Preferred approach:

```text
Use retailer buttons instead of static prices.
```

## Library Access Rules

Only mention Hoopla or library borrowing if verified.

Do not guarantee library access.

Preferred wording when verified:

```text
Readers with access to participating libraries may be able to borrow this title digitally through Hoopla.
```

Avoid:

```text
Borrow this book for free from every library.
```

## Goodreads Rules

Goodreads may include duplicate entries, alternate editions, or user-generated content.

Use Goodreads for:

* Author profile links
* Book discovery
* Edition cross-checking
* Reader follow CTA

Do not use Goodreads to create fake review language.

Do not quote Goodreads reviewers unless:

1. The quote is verified
2. The client approves its use
3. The use complies with platform and copyright considerations

Do not claim “actively interacts with reviews” unless directly confirmed and approved.

## Review and Testimonial Rules

No fake reviews.

No fake testimonials.

No invented reader praise.

Do not create generic praise such as:

```text
Readers are calling it unforgettable.
Critics praise Eric’s explosive storytelling.
Fans can’t stop talking about it.
A must-read fantasy epic.
An award-worthy thriller.
```

Only use praise if:

1. It is verified from an approved source
2. It is provided by the client
3. It is approved for website use

## Video and Media Rules

Do not invent video URLs.

Do not invent trailer lengths.

Do not invent video titles.

Do not embed social videos unless the exact URL is confirmed.

If the exact URL is missing, use an internal TODO.

Acceptable internal note:

```text
TODO: Add verified YouTube URL for the Rage of Angels trailer.
```

Not acceptable visible site copy:

```text
Watch the official cinematic trailer here.
```

Unless the trailer URL is actually embedded or linked.

## Metadata Rules

Metadata must not contain fake claims.

Do not write metadata like:

```text
Award-winning author Eric Deloach
Bestselling Rage of Angels series
Internationally acclaimed thriller author
Critically praised fantasy epic
```

Use verified, safe metadata instead:

```text
Explore books by Eric Deloach, author of the Rage of Angels universe and In Harm’s Way. Discover dark fantasy, gritty action, trailers, and retailer links.
```

## Schema Rules

Structured data must not include fake facts.

Do not include unverified fields such as:

* awards
* aggregateRating
* review
* offers with price
* publisher
* narrator
* datePublished
* numberOfPages
* isbn

Unless each value is verified and approved.

If a field is uncertain, omit it.

## Event Rules

Do not add book signings, appearances, release parties, panels, readings, or events unless explicitly verified and approved.

If an events page exists before events are confirmed, use neutral copy:

```text
Upcoming events and appearances will be shared here when available.
```

Do not invent an event calendar.

## Newsletter Rules

Do not claim a newsletter exists unless the client has a signup provider or approved mailing list workflow.

Acceptable planning language:

```text
Join the Reader List
```

Do not publish a live signup form unless the destination or email workflow is configured.

## Contact Rules

Do not publish personal contact details unless approved.

If no public contact email is approved, use a contact form or placeholder workflow only in planning, not visible live copy.

## Image Rules

Do not use fake author portraits.

Do not use AI-generated images of Eric unless the client explicitly approves.

Do not use book cover recreations unless provided or approved.

Do not use military imagery that falsely implies real military endorsement.

Do not use logos from retailers unless permitted or implemented according to brand guidelines.

## Design Copy Rules

Do not let design language create false claims.

Avoid:

```text
The bestselling saga
Award-winning author
International sensation
Critics agree
Millions of readers
As seen on
Official Navy story
True story
Based on real events
```

Unless verified and approved.

## Content Lock Rule

Once content is approved, Claude Code must not rewrite it.

Locked content includes:

* Headlines
* Section copy
* Book descriptions
* Author bio
* CTAs
* FAQs
* Metadata
* Schema facts
* Slugs
* Navigation labels
* Retailer labels

Any change to locked content requires explicit approval.

## Handling Uncertainty

When uncertain, use one of these internal labels:

```text
Needs verification
Do not publish yet
Client-provided, confirm before launch
Verified, safe to publish
Approved for planning only
```

Do not display these labels to users on the live website unless they are part of an internal admin or project doc.

## Developer Notes

Claude Code may leave TODO comments in source files for missing data.

Acceptable:

```text
// TODO: Add verified Apple Books Audiobook URL for In Harm’s Way.
```

Unacceptable:

```text

```

Never ship visible placeholder links.
